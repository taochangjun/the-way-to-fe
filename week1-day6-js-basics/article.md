# 别急着学 DOM：后端转前端，我把 Day6 改成了“先 JS 基础”

> 如果你也在学前端，并且总觉得“DOM API 看得懂但写不顺”，这篇会帮你省下很多绕路时间。

我最近在做一条前端学习路线，原本 Day6 是 DOM 操作。  
但练下来发现一个典型问题：**DOM 调用会写，逻辑却不稳**。  

于是我把 Day6 重构成了：**只练 JavaScript 基础语法与数据类型**，先不讲 DOM 深水区。

## 1. 为什么“先 JS 再 DOM”更顺

先说结论：  
**DOM 是浏览器提供的 Web API，不属于 ECMAScript 核心语法。**

你在写 DOM 代码时，真正的难点通常不是 `querySelector`，而是：

- 条件分支写不清
- 循环边界容易错
- 函数拆分混乱
- 数组数据处理绕来绕去

换句话说，DOM 是“操作页面的手”，JS 基础是“思考逻辑的脑”。  
脑子没练稳，手法就会越写越乱。

## 2. Day6 具体练什么（不碰复杂 DOM）

我把 Day6 定成 4 个小练习，全部是纯 JS 逻辑：

1. **成绩等级判断**（条件分支）  
2. **1-100 偶数求和**（循环）  
3. **订单汇总对象**（函数 + 对象）  
4. **价格处理**（`map/filter/reduce` 预热）

对应代码非常基础，比如等级判断：

```js
function getGrade(score) {
  if (score >= 90) return "A";
  if (score >= 80) return "B";
  if (score >= 70) return "C";
  if (score >= 60) return "D";
  return "F";
}
```

看着简单，但这一步很关键：  
你开始从“写一坨脚本”转向“写可复用函数”。

## 3. 我用的 2 小时训练法（可直接套用）

如果你时间不多，可以按这个节奏：

- **30 分钟：概念输入**  
  看变量、条件、循环、函数、数组方法，不求全懂，先有印象。

- **60 分钟：代码输出**  
  做 4 个小练习，每题都强制“先写函数，再调用函数”。

- **20 分钟：复盘**  
  记录：今天最容易错的 3 个点 + 明天怎么避免。

- **10 分钟：补洞**  
  把 `reduce`、类型转换、边界判断再过一遍。

> 你会明显感到：写代码不再“靠感觉”，而是有步骤地完成。

## 4. 常见误区（我自己踩过的）

### 误区 1：过早沉迷 API

一上来就刷 DOM、刷框架 API，会产生“我会很多方法”的错觉。  
一到业务逻辑就卡住。

### 误区 2：把逻辑写死在交互回调里

比如所有计算都塞进点击事件。短期能跑，后期很难维护。  
建议先函数化，再接页面。

### 误区 3：忽略类型转换

输入框拿到的是字符串，忘记 `Number(...)` 迟早出 bug。

## 5. 学到什么程度就够了？

如果你的目标是 Vue 3 / 全栈应用开发，Day6 的目标不是“成为 JS 算法高手”，而是：

- 能稳定写出条件 + 循环 + 函数
- 能用对象表达业务数据
- 能看懂并使用 `map/filter/reduce`
- 知道如何把逻辑拆小并复用

达到这一步，再进入 Day9-Day10 的 DOM 与事件，会顺很多。
