# Day6 学习文档：JS 基础语法与数据类型

## 为什么 Day6 先学 JS，再学 DOM

DOM 属于浏览器提供的 Web API，不是 JavaScript 核心语法本身。  
如果先把 JS 基础打牢，再进入 DOM，你会更容易理解“数据如何变成页面交互”。

这就是当前 Day6 的目标：**只练逻辑，不讲 DOM 操作细节**。

## 关键知识速记

### 1) `const` 与 `let`

- 默认优先 `const`
- 需要重新赋值时用 `let`

### 1.1) `var` 与 `let` 对比（面试和实战高频）

- **作用域**：`var` 是函数作用域，`let` 是块级作用域（`if/for/{}` 内有效）
- **变量提升**：`var` 会提升并初始化为 `undefined`；`let` 也会提升但处于暂时性死区（TDZ），声明前访问会报错
- **重复声明**：同一作用域内 `var` 可以重复声明，`let` 不允许重复声明
- **全局对象挂载**：全局 `var a = 1` 会挂到 `window.a`；`let a = 1` 不会挂到 `window`

示例（最常见坑：循环闭包）：

```js
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log("var:", i), 0); // 3, 3, 3
}

for (let j = 0; j < 3; j++) {
  setTimeout(() => console.log("let:", j), 0); // 0, 1, 2
}
```

### 2) `===` 与 `==`

- 日常优先 `===`，避免隐式类型转换带来的误判

### 3) 先函数化，再接页面

推荐习惯：先把业务逻辑写成函数并验证，再放到页面交互中调用。  
这会让你后面学 Vue 时非常顺手。

## 常见坑

1. 输入值没转数字就直接计算（需要 `Number(...)`）
2. 循环边界写错（`<=` 和 `<` 混用）
3. `reduce` 忘记初始值
4. 逻辑全写在事件回调里，函数无法复用
